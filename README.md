# 🔐 Browser Automation — Headless Login & Session Persistence

مشروع تعليمي/بورتفوليو يوضّح استخدام **Playwright (Python)** لأتمتة تسجيل الدخول وإدارة الجلسة (Session) في وضع **Headless** بالكامل — بدون فتح أي نافذة متصفح مرئية، وهو الوضع المستخدم فعلياً في بيئات الإنتاج (سيرفرات، Cron Jobs، أتمتة دورية).

## ⚙️ ماذا يفعل السكربت؟

1. أول تشغيل: يفتح متصفح Headless، يسجّل دخول حقيقي على [saucedemo.com](https://www.saucedemo.com/) (موقع تجريبي رسمي مخصص لممارسة أتمتة المتصفح)، ثم يحفظ الجلسة (`auth_state.json`).
2. أي تشغيل تالي: يعيد استخدام الجلسة المحفوظة مباشرة — **بدون** المرور بنموذج تسجيل الدخول مرة ثانية.
3. يتحقق من صحة الجلسة عبر قراءة عدد الكوكيز، الرابط الحالي، واسم أول منتج بالصفحة المحمية.

## 🧠 الدروس المستفادة (الجزء الأهم)

هذا القسم يوثّق مشاكل حقيقية واجهتها أثناء التطوير وكيف تم حلها — مو بس "كود شغال":

### 1. الصفحة الرئيسية لا تعكس حالة الجلسة دائماً
كنت أتوقع إن `page.goto("https://www.saucedemo.com/")` مع جلسة صالحة يوديني مباشرة للصفحة المحمية. الواقع: هذا الموقع تحديداً يعرض نموذج تسجيل الدخول دايماً على الصفحة الرئيسية، بغض النظر عن الجلسة. **الحل:** التنقّل مباشرة إلى الصفحة المحمية (`/inventory.html`) بدل الاعتماد على إعادة توجيه تلقائي — درس عام: لا تفترض سلوك الموقع، تحقق منه فعلياً.

### 2. عمليات Chromium المعلّقة تسبب تجمّد الطرفية
بعض المرات كانت عملية Chromium الفرعية ما تُغلق بشكل نظيف، فيعلّق `Ctrl+C` بدون استجابة. **الحل العملي:**
```
taskkill /F /IM chrome.exe /T
taskkill /F /IM python.exe /T
```

### 3. تلف الترميز (Mojibake) عند التعديل بمحرر عادي
تعديل الكود جزئياً بـ Notepad العادي (بدون ضمان UTF-8) أدى لتلف بعض الأحرف. **الحل:** عند الشك بالترميز، استبدال الملف بالكامل بنسخة UTF-8 نظيفة بدل الترقيع الجزئي.

### 4. الفرق بين Demo محلي (`file://`) وموقع حقيقي
على ملف HTML محلي، عدد الكوكيز = صفر دائماً لأنه ما فيه سيرفر حقيقي يرسل `Set-Cookie`. التحقق الحقيقي من عمل الجلسة يحتاج موقع فعلي (لهيك استخدمت saucedemo.com بدل الديمو المحلي بهذه المرحلة).

## 🛠️ التقنيات المستخدمة

| التقنية | الاستخدام |
|---|---|
| Python 3.10+ | لغة التطوير |
| Playwright (sync API) | التحكم بالمتصفح (Headless Chromium) |
| `storage_state()` | حفظ/استرجاع الجلسة (كوكيز + localStorage) |
| `wait_for_selector` / `wait_for_url` | استراتيجيات انتظار ذكية بدل `time.sleep()` |

## 🚀 التشغيل محلياً

```bash
# 1. إنشاء بيئة افتراضية
python -m venv venv
venv\Scripts\activate          # Windows
# source venv/bin/activate     # macOS/Linux

# 2. تثبيت المتطلبات
pip install -r requirements.txt
playwright install chromium

# 3. تشغيل السكربت
python stage3_playwright.py
```

**أول تشغيل** يسجّل دخول حقيقي ويحفظ الجلسة. **أي تشغيل بعده** يستخدم الجلسة المحفوظة مباشرة (احذف `auth_state.json` لإعادة تسجيل الدخول من الصفر).
